# Техническая документация на ендпоинты категории **profiles**

## GET /profiles/

### Описание
Данный ендпоинт выдаёт список профилей пользователей, прошедших все этапы процедуры регистрации.

### Требования
Запрашивающий пользователь должен относиться к любой группе авторизованных пользователей (guest, user, moderator, admin, etc).

## GET /profiles/{id}/

### Описание
Данный ендпоинт выдаёт профиль пользователя по уникальному идентификатору `id`, если запрашиваемый пользователь прошёл все этапы процедуры регистрации.

Успешный запрос вернёт следующую информацию о пользователе:
1. ФИО, профессию;
2. Локацию (город и страна проживания, возможность переезда);
3. Образование (уровень, название учебного заведения, год окончания);
4. Описание (произвольное краткое и полное описание);
5. Ссылки на социальные сети (телеграмм, github, vk);
6. Изображения (аватор, фотообои).
7. Предпочитаемые форматы работы.

### Требования
Запрашивающий пользователь должен относиться к любой группе авторизованных пользователей (guest, user, moderator, admin, etc). Это самая нестрогая стратегия **открытости профиля**.

Запрашивающий пользователь не должен быть в чёрном списке запрашиваемого пользователя. Это стратегия **открытости для порядочных пользователей**.

Запрашивающий пользователь должен быть в белом списке запрашиваемого пользователя, в случае, если в профиле запрашиваемого пользователя для **всех** полей настроена политика конфиденциальности на уровне **Невидно никому, кроме...**. Если есть хотя бы одно более открытое поле (**Видно всем** или **Видно всем, кроме...**), то для запрашивающего пользователя не из белого списка будут видны **только открытые поля**. Это избирательная стратегия **открытости для избранных/друзей/коллег/посторонних и т.д.**.

Профиль запрашиваемого пользователя не будет виден никому, если **все** поля профиля будут закрыты на уровне **Невидно никому**. Это самая строгая стратегия **закрытости профиля**.

## POST /profiles/{id}/

### Описание
Для первичного создания и последующего обновления обновления профиля зарегистрированного пользователя.

Для заполнения доступны следующие общие поля профиля:
1. Персональные данные (ФИО, профессия);
2. Локация (город и страна проживания, возможность переезда);
3. Образование (уровень, название учебного заведения, год окончания);
4. Описание (произвольное краткое и полное описание);
5. Ссылки на социальные сети (телеграмм, github, vk);
6. Изображения (аватор, фотообои);

В случае успеха при полном или частичном обновлении профиля возвращается контент с полным профилем, как и при GET запросе.

### Требования
- Изменение возможно только собственного профиля (группа **permissions - IsAccountOwner**).

### Поля запроса
| Поле | Тип | Обязательное | Значение по умолчанию | Описание |
|------|-----|--------------|-----------------------|----------|
| `first_name` | string | Нет | `''` | Имя пользователя |
| `middle_name` | string | Нет | `''` | Отчество пользователя |
| `last_name` | string | Нет | `''` | Фамилия пользователя |
| `profession` | string | Нет | `''` | Профессия |
| `city` | string | Нет | `''` | Город проживания |
| `country` | string | Нет | `''` | Страна проживания (гражданство) |
| `relocation` | string | Нет | `''` | Место для переезда |
| `edu_level` | string | Нет | `'nothing'` | Уровень образования |
| `institution_name` | string | `''` | Нет | Наименование учебного заведения |
| `graduation_year` | integer | `null` | Нет | Год окончания учёбы |
| `short_desc` | string | Нет | `''` | Произвольное краткое описание |
| `full_desc` | string | Нет | `''` | Произвольное подробное форматированное описание |
| `wallpaper` | string | Нет | `null` | Ссылка на обои пользователя (не работает в swagger UI) |
| `avatar` | string | Нет | `null` | Ссылка на аватар пользователя (не работает в swagger UI) |
| `link_to_instagram` | string | Нет | `''` | Ссылка на инстаграмм (запрещен в РФ) |
| `link_to_telegram` | string | Нет | `''` | Ссылка на телеграм |
| `link_to_github` | string | Нет | `''` | Ссылка на github профиль |
| `link_to_vk` | string | Нет | `''` | Ссылка на вк-профиль |

Изменение данных профиля проводится и в других ендпоинтах:
- POST /profiles/{profile}/skills/
- PATCH /profiles/{profile}/skills/{skill}/
- PUT /profiles/{profile}/skills/{skill}/privacies/
- PATCH /profiles/{profile}/work_formats/

## DELETE /profiles/{id}/

### Описание
Удалить можно только собственный профиль. По факту производится мягкое удаление (деактивация) профиля. Полное удаление профиля доступно по запросу пользователя через форму обратной связи.

Удаление чужого профиля пользователя (со всеми связанными данными) доступно пользователям из групп admin и moderator.

## GET /profiles/{profile}/skills/

### Описание
Данный ендпоинт выдаёт список всех навыков пользователя, которые он указал при заполнении профиля.

Успешный запрос вернёт список навыков, каждый из которых будет содержать следующую информацию:
1. Уникальный идентификтор навыка пользователя;
2. Информация о навыке (глобальная);
   1. Глобальный идентификатор навыка;
   2. Нименование навыка;
   3. Описание;
   4. Даты создания и обновления;
3. Персональный уровень владения навыком;
4. Тип навыка (активный/спящий);
5. Настройка конфиденциальности;
6. Ссылки на социальные сети (телеграмм, github, vk);
7. Даты создания и обновления;

### Требования
Запрашивающий пользователь должен соответствовать требованиям, предъявленным в ендпоинте `GET /profiles/{id}/`.

То есть увидит ли запрашивающий пользователь данные о навыках зависит от стратегии и настроек конфиденциальности запрашиваемого пользователя.

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

## POST /profiles/{profile}/skills/

### Описание
Для добавления уникального навыка в свой профиль.

При добавлении на стороне бекэнда проверяется наличие указанного навыка в глобальном списке навыков. При отсутствии указанного пользователем навыка в глобальном списке происходит его автоматическое добавление (это нужно, чтобы в будущем предлагать пользователям уже существующие навыки).

После добавления навыка в свой профиль пользователь уже не сможет изменить его наименование (нужно будет добавлять новый навык).

В случае успеха возвращается контент с данными о навыке.

### Требования
- Добавление навыка возможно только в собственный профиль (группа **permissions - IsAccountOwner**).

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

### Поля запроса
| Поле | Тип | Обязательное | Значение по умолчанию | Описание |
|------|-----|--------------|-----------------------|----------|
| `skill_name` | string | Да | - | Наименование навыка |
| `level` | integer | Нет | `1` | Уровень владения навыком (от 1 до 10) |
| `is_current` | boolean | Нет | `true` | Актуальность навыка (активный/спящий) |
| `privacy` | string | Нет | `?` | Настройка конфиденциальности (значение по умолчанию зависит от выбранной стратегии) |

## GET /profiles/{profile}/work_formats/

### Описание
Данный ендпоинт позволяет пользователю выбрать предпочитаемые форматы работы (в офисе, гибрид или удалённо).

Успешный запросе вернёт контент с предпочтениями:
1. **id** - уникальный идентификатор, свойственный для конкретного пользователя.
2. **office** — `true`, если выбран данный формат, `false`, если нет.
3. **hybrid** — `true`, если выбран, `false`, если нет.
4. **remote** — аналогично.

### Требования
Запрашивающий пользователь должен соответствовать требованиям, предъявленным в ендпоинте `GET /profiles/{id}/`.

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

## PATCH /profiles/{profile}/work_formats/

### Описание
Для изменения предпочитаемых форматов работы. Можно выбрать что-то одно или комбинацию нескольких.

Успешный запрос вернёт текущие предпочтения.

### Требования
- Изменение доступно только для собственного профиля (группа **permissions - IsAccountOwner**).

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

### Поля тела запроса
| Поле | Тип | Обязательное |Описание |
|------|-----|--------------|---------|
| `office` | boolean | Нет | Работа из офиса (`true` - выбрано) |
| `hybrid` | boolean | Нет | Гибридная работа (`true` - выбрано) |
| `remote` | boolean | Нет | Удалённая работа (`true` - выбрано) |

## GET /profiles/displays/skills/

### Описание
Данный ендпоинт возвращает список уникальных навыков, созданных всеми пользователями ресурса.

Нужен для того, чтобы предлагать новым пользователям формулировки навыков, которые уже существуют в системе, возможно, это то, чем обладает пользователь.
