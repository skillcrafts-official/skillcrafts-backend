# Техническая документация на ендпоинты категории **privacy settings**

## GET /privacy-settings/profiles/{profile}/privacies/

### Описание
Данный ендпоинт позволяет клиенту получать настройки приватности профиля пользователя.

Успешный запрос вернёт контент с настройками конфиденциальности собственного профиля. Настройки содержат уровни досутпа к каждому полю профиля пользователя (см. ендпоинт **GET /profiles/{id}/**). Дополнительно будут возвращены чёрный и белый списки пользователей.

### Требования
- Предварительно, пользователь должен пройти процедуру аутентификации в клиенте, с которого запрашиваются настройки конфиденциальности, иначе получить данные невозможно, включая группы пользователей `admin` и `moderator`.

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор авторизованного в клиенте пользователя |

## PATCH /privacy-settings/profiles/{profile}/privacies/

### Описание
Для гибкого формирования, управления и реализации политики конфиденциальности. Для каждого поля профиля можно задавать персональную политику.

Успешный запрос установит новые настройки политики конфиденциальности для указанных в теле запроса полей и вернёт текущие настройки для профиля пользователя, включая чёрный и белый списки.

Для каждого поля доступны следующие значения на выбор:
- **all** — соответствует политике **Видно всем** авторизованным или неавторизованным пользователям;
- **not_all** — соответствует политике **Видно всем, кроме...** (кроме неавторизованных пользователей и тех, кто в чёрном списке);
- **no_one_except** — соответствует политике **Невидно никому, кроме...** (кроме авторизованных пользователей в белом списке текущего пользователя);
- **nobody** — соответствует политике **Невидно никому** (ни один клиент не получит данные отмеченные данной настройкой, даже группы пользователей admin и moderator).

Политика конфиденциальности реализуется за счёт маскирования данных, то есть данные пользователя остаются на сервере, а отправляются макси.

### Требования
- Изменение доступно только для собственного профиля (группа **permissions - IsAccountOwner**).

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

### Поля тела запроса
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `first_name` | string | Нет | Имя пользователя |
| `middle_name` | string | Нет | Отчество пользователя |
| `last_name` | string | Нет | Фамилия пользователя |
| `profession` | string | Нет | Профессия |
| `city` | string | Нет | Город проживания |
| `country` | string | Нет | Страна проживания (гражданство) |
| `relocation` | string | Нет | Место для переезда |
| `work_formats` | string | Нет | Уровень образования |
| `edu_level` | string | Нет | Уровень образования |
| `institution_name` | string | Нет | Наименование учебного заведения |
| `graduation_year` | integer | Нет | Год окончания учёбы |
| `short_desc` | string | Нет | Произвольное краткое описание |
| `full_desc` | string | Нет | Произвольное подробное форматированное описание |

## PATCH /privacy-settings/profiles/{profile}/blacklist/

### Описание
Ендпоинт предназначен для добавления пользователей в чёрный список или удаления из него.

Успешный запрос вернёт статус операции (поле `success`) и обновлённый чёрный список текущего пользователя. Список будет состоят из уникальных идентификаторов пользователей.

Для каждого поля запроса доступны следующие значения на выбор:
- **action** — может принимать значения `block` или `unblock` (для блокировки или разблокировки пользователя по **id**);
- **for_user_id** — уникальный идентификтор пользователя, которого нужно либо добавить в чёрный списко, либо удалить из него.

### Требования
- Управление чёрным списком должно быть доступно только для собственного профиля (группа **permissions - IsAccountOwner**).

### Параметры пути
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `profile` | integer | Да | Уникальный идентификатор запрашиваемого пользователя |

### Поля тела запроса
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `action` | string | Да | Операция над чёрным списком |
| `for_user_id` | integer | Да | Уникальный идентификатор пользователя |
